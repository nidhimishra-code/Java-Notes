Spring Boot Interview Questions 


1. What do you understand by Spring Boot?

ans. Spring Boot is an open-source Java-based framework that simplifies the development of Spring applications by eliminating boilerplate configurations.
It also offers embedded servers, default configurations, and starter dependencies.

2. How does Spring Boot differ from the traditional Spring Framework?

ans. It will handle all configuration.

3. What are the main features of Spring Boot?

ans. Auto-configuration - give port number 
     Spring Boot Starters
     Spring Boot Actuator - give health of website 
     Spring Boot Initializr
     

4. What are Spring Boot starters ?

ans. Web Application
        Spring Web - Embedded web server (Tomcat), Spring MVC, Jackson, Support for building RESTful web services
        Spring Boot DevTools
   
     Database
        Spring Data JPA
        H2 Database
        MySQL or PostgreSQL

     Security
        Spring Security


    These are pre-configured sets of dependencies designed to simplify the setup and configuration of commonly used technologies and functionalities.

    Ensures compatibility and reduces the chances of missing critical dependencies.


5. How do you create a Spring Boot application using spring Initializr?

ans. Spring Initializr - A online tool that generates a skeleton Spring Boot project with the necessary configurations and dependencies.


6. What is the purpose of the @SpringBootApplication annotation?

ans. @Configuration - Declares the class as a source of Spring configuration.
     @EnableAutoConfiguration - Enables Spring Boot's auto-configuration
     @ComponentScan - Scans for Spring-managed components in the package.


7. Is it possible to change the port of the embedded Tomcat server in Spring Boot?

ans. By using the server.port in the application.properties.


8. Can we override or replace the Embedded tomcat server in Spring Boot?

ans. Yes, we can replace the Embedded Tomcat server with any server by using the Starter dependency in the pom.xml file.


9. What is auto-configuration in Spring boot? how does it help? Why Spring Boot is called opinionated?

ans. Autoconfiguration is spring boot magic features, it automatically configures a lot of things based upon what is present in the class path.

Spring Boot is called opinionated because it does configuration by itself.


10. What is the difference between @SpringBootApplication and @EnableAutoConfiguration annotation?

ans. @EnableAutoConfiguration is used to enable auto-configuration but @SpringBootApplication does a lot more than that.

It also combines @Configuration and @ComponentScan annotations to enable Java-based configuration and component scanning in your project.

@SpringBootApplication = @Configuration + @EnableAutoConfiguration + @ComponentScan


11. How to disable a specific auto-configuration class?

ans. We can use exclude attribute of @EnableAutoConfiguration(exclude={className}) annotation to exclude specific class.


12. What is the difference between @RestController and @Controller in Spring Boot?

ans. @Controller - Map of the model object to view or template and make it human readable 

@RestController - simply returns the object and object data is directly written in HTTP response as JSON or XML

       @RestController = @Controller + @ResponseBody


13. Describe the flow of HTTPS requests through the Spring Boot application?

               HTTPS
      Client <-------> Controller <------> Service layer <-----> Model
               Request                         ^                  ^
                                               |                  |
                                                ----> Database <---
                                                                 JPA/Spring
                                                                    Data




14. What is the difference between @RequestMapping and @GetMapping?

ans. @RequestMapping can be used with GET, POST, PUT, and many other request methods using the methods attribute on the annotation.

Whereas @GetMapping is only an extension of @RequestMapping with GET method which helps you to improve on clarity on request.

@RequestMapping(value = "/user/{userId}", method = RequestMethod.GET)

                 @GetMapping(value = "/user/userId}")



15. What is the difference between an embedded container and a WAR?

ans. The main difference between an embedded container and a WAR file is that you can run Spring Boot application as a JAR from the command prompt without setting up a web server.

But to run a WAR file, you need to first set up a web server like Tomcat which has Servlet container and then you need to deploy WAR there.


16. What is the use of Profiles in spring boot?

ans. Spring has the provision of Profiles to keep the separate configuration of environments.


17. What is Spring Actuator? What are its advantages?

ans. Provides special feature to monitor and manage your application when you push it to production.

18. How to get the list of all the beans in your Spring boot application?

ans. Spring Boot actuator "/Beans" is used to get the list of all the spring beans in your application

"/env" returns the list of all the environment properties of running the spring boot application.
